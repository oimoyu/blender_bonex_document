序章

BoneX は、PhysX によって駆動される骨物理アニメーションの作成を支援する Blender アドオンです。 PhysX は、UE4、U3D などのゲーム分野で使用される成熟した物理エンジンです。 PhysX を利用して安定した剛体シミュレーションを生成すると、キャラクター アニメーション用のボーン物理効果を作成できます。 剛体のプロパティを変更し、その接続の剛性、減衰品質などを設定して、必要な物理的効果を実現できます。

Platform Support: Windows

Blender Version: 3.6+

UI Language Support: English, 日本語, 中文

これを開発した理由

鮮やかな物理学により、キャラクター アニメーションに比類のないレベルのリアリズムと深みがもたらされます。 Blender を使用する場合、これを実現するためのクロスや剛体のシミュレーションなど、多くの方法が存在します。ゲーム エンジンの分野では、剛体シミュレーションはキャラクターの物理学によく使用されます。 Blender の剛体シミュレーションに基づいた無料のアドオン blender_スカート_rigid_generator を作成しようと努力しましたが、まだ多くの問題があります。 その結果、私は、安定した高速な結果を提供する、成熟した信頼性の高い PhysX 物理エンジンを活用して、これを開発することにしました。

利点

①成熟物理学エンジンPhysxでは、シミュレーションは安定しており、ジッターが少なくなります

② コリジョン グループ マスクを使用してコリジョンの相互作用を制御することがサポートされています

③カスタム コリジョン メッシュがサポートされています

④リアルタイム再生インタラクション

⑤部分シミュレーションに対応し、細かな調整を改善します

etc.

スカートのシミュレーションに布地シミュレーションではなくリジッド ボディを使用するのはなぜですか?

① 安定して制御可能。 剛体シミュレーションは、限られた数の剛体に焦点を合わせるだけです。

② 形を保つ。スタイライズされたキャラクターを作る場合、物理法則に反するような大げさな形を作る必要があります。布シミュレーションを使うと、このような形状は布の崩壊とともに崩れてしまいます。剛体シミュレーションはボーンを駆動し、1つのボーンが多数の頂点を制御できます。ボーンシミュレーションは、ボーン間の相対的な位置関係を維持します。

③ メッシュの交差を防ぐ。プロキシに布シミュレーションを使用する場合、接触とメッシュの交差が問題となる。布のコリジョンボリュームは薄いレイヤーでしかないが、剛体のコリジョンボリュームは自分で定義できる。フワフワしたメッシュの場合、コリジョンボリュームを大きくすることで、他のメッシュがモールドと交差するのを防ぐことができる。

GPU

通常、これは必要ありません。パフォーマンスのボトルネックは physx にはなく、(私のテストでは) 逆効果を引き起こす可能性があります。PhysxはCUDAハードウェアアクセラレーションをサポートします。 多くの剛体がある場合、または反復数が大きく、フレーム生成が長い時間がかかる場合、GPUを使用すると計算速度が大幅に増加します。 GPUアクセラレーション関数を使用する必要がある場合は、「PhysXGPU_64.DLL」と「PhysXDevice64.DLL」をダウンロードし、Addonディレクトリの「Physx.exe」と同じ場所に配置する必要があります。

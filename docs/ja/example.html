パラメータと例

パラメータのヒント

開始前: 剛体 パラメータはリアルタイムではありません。パフォーマンスを消費するためです。 Rigidbody パラメータの変更結果を確認するには、再生を再開する必要があります。

パラメータ調整テクニック、以下のテクニックは他の物理エンジンにも適用できます

チェーン上の剛体については、それらの間にバネを置いて 2 つずつ接続するのと同じです。 希望の結果が得られるように、適切なパラメータを調整する必要があります。

①1つ目は形状を維持することです。 剛体チェーンは重力の影響によりたわみます。 重力によってたるみが生じたくない場合は、重力を 0 に調整するか、質量を減らすか接続の剛性を高めます (1 つのパラメーターの変更が他の側面に影響するため、パラメーターの配置に注意してください) 変更)

②質量とバネ剛性の関係。 剛体チェーンが揺れると、スプリング接続の力と剛体の慣性力の影響を受けます。 質量が大きいほど慣性力が大きくなり、揺れが良くなります。 バネの剛性が大きいほど、バネは変形するのに必要な力が大きくなり、揺れにくくなります。 たとえば、鉄片がバネに結び付けられ、一枚の紙がバネに結び付けられていると想像してください。 どっちがよく揺れますか？ バネの剛性が大きいほど反発力も大きくなることに注意してください。 そのため、剛性の大きなバネは揺れると初期位置を中心に前後に揺れてしまいます。

③バネの剛性と減衰の関係。ダンピングとは、スプリングが元の位置に戻らないようにする力のことです。 ダンピングが大きいほど抵抗も大きくなります（抵抗はスプリングの反発力を超えません）。 バネの剛性によるリバウンドを遅くし、剛体が初期位置を中心に前後に揺れたくない場合、またはモーションを少し柔らかく見せたい場合に必要なパラメータです。

質量と運動の関係。質量が大きいほど慣性力は大きくなる。物体が宙に浮いている感じを表現したいのであれば、逆に質量を大きくする必要がある。質量が小さい剛体はバネの硬さに引っ張られてすぐに戻りやすく、宙に浮いている時間は長くない。質量が大きい物体は慣性力が大きく、同じバネの硬さでも引っ張るのに時間がかかるので、宙に浮いている感じを反映できる。

⑤スケールは重要です。 シミュレーションは物理法則に従って実行されます。 たとえば、胸の骨に剛体を与えると、上下に振ることができない場合があります。 スケーリングが小さすぎてボーンが短すぎたり、重心モーメント アームが短すぎたりする可能性があるかどうかを考慮する必要がある場合があります。 この場合、モデルを拡大してみることもできますが、これは多くの場合面倒なので、剛体の質量を増やすか、剛性を下げることを試みることができます。

一般的にはねじる必要がないため、ねじり軸方向の角度制限を0に設定できます

例

サンプルビデオチュートリアルは順次更新されます。

毛皮のような耳としっぽ

テールの場合は、跳ね返るように中程度の剛性が必要で、より自然に見えるように中程度の減衰が必要です。 耳についてはテールの設定を参照できますが、ソフト効果を反映するためにより高いダンピングを与えることができます。

髪

髪の場合、髪は非常に柔らかいため、剛性は低くする必要があり、髪は非常に軽く、空気抵抗が大きいため、減衰は高くする必要があります。 また、バウンス係数を小さくすると、髪が柔らかくて軽くなり、現実世界では髪が他のものと衝突してもあまり跳ねなくなります。

布

布の場合は髪のパラメータを参照し、重力や剛性を上げてください

スカート ボーンに関するヒント。スカート ボーンのロールが正しくない場合は、編集モードでボーンを選択し、「Shift+N -> カーソル」を実行します (完璧ではないかもしれませんが、手動で編集するよりも高速です)。

胸

胸の場合、一定の揺れ効果を生み出し、弾力性を反映するには、わずかに高い剛性が必要です。 同時に、変位制限を 0 よりわずかに大きく調整すると、より良い物理的効果を得ることができます。

シンプルなラグドール

dynaimc剛体は、kinematic剛体の親ボーンに表示されないことに注意してください。 Blender では子ボーンの変形は親ボーンの影響を受けますが、PhysX ではキニマティック リジッド ボディの動きがダイナミック リジッド ボディに影響を与えるため、周期的な効果が発生してシミュレーション結果が台無しになってしまいます。 それでもそのような効果を作成したい場合は、尾部に設定されたキニマティック リジッド ボディに「コピー変換」ボーン コンストレイントを使用して、その変位と回転が親ボーンの影響を受けないようにする必要があります(connectedがチェックされていないことに注意。そうでないと変位がロックされる)。または、ソフトコネクションを使用してこれを取得することもできます。
